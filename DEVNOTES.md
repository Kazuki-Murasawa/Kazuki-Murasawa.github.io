## プロジェクト概要

- Firebaseを使った社内専用Web Wikiシステム（`index.html` 1ファイル構成）
- 元はToDoアプリだったが、Firestoreのコレクション名を `wiki_pages` に変更し、Wikiシステムへ移行済み
- 2026-07-05: UIをスマホ向けTodoリスト風から、PCブラウザ前提のWikipedia/PukiWiki風
  「サイドバー＋一記事表示」レイアウトに全面刷新（下記「現在のUI構造」参照）

## 重要な制約（崩さないこと）

- JavaScriptは `<script type="module">` で動いている
- `marked` はCORS制限回避のため `<head>` 内の通常の `<script src="...">` タグで
  グローバル読み込みし、モジュール内で `const marked = window.marked;` として紐付けている
  → このロード方式を変更しない
- Firestoreの `wiki_pages` コレクションはリアルタイム購読（`onSnapshot`）。取得したデータは
  モジュール変数 `currentPages` にキャッシュし、画面状態（表示中ページ・編集中・新規作成中・
  検索文字列・タグ絞り込み）も同様にモジュール変数で管理している

## 現在のUI構造（2026-07-05〜）

- **ヘッダー**: ブランド（クリックでホームへ `#/`）＋ ログイン/ユーザー情報
- **サイドバー（左・sticky）**: 検索ボックス、「＋ 新規ページ」「＋ 新規フォルダ」ボタン、
  ページ一覧（フォルダ階層＋ページのツリー表示）、タグ絞り込みバッジ
- **メインエリア（右）**: 状態に応じて以下のいずれかを描画（`renderMain()`）
  - ホーム（未選択時）: ウェルカムメッセージ＋最近更新されたページ5件
  - 記事詳細（1記事のみ表示、Wikipedia風）: タイトル・タグ・Markdown本文・投稿者/日時・
    投稿者本人のみ表示される編集/削除ボタン
  - 新規作成フォーム／編集フォーム（共通テンプレート `renderPageFormHtml`）
- **ルーティング**: `location.hash` で管理（`#/page/<firestoreドキュメントID>` / `#/new` / 空でホーム）。
  `hashchange` イベントで状態を切り替えるので、URLをブックマーク・共有すれば該当ページに直接飛べる
- 旧仕様にあった「全記事を1ページに連続表示するフィード」「目次(TOC)アンカー」は、
  この構造では不要になったため撤去済み（役割はサイドバーのページ一覧に統合された）

## 実装済み機能

- Googleログイン / ログアウト
- 記事の投稿・閲覧・編集・削除（Markdown対応、投稿者本人のみ編集・削除可）
- タグ機能（カンマ区切りで登録、`tags` 配列としてFirestoreに保存）。サイドバーのタグバッジで
  ページ一覧を絞り込み（検索文字列とAND条件）
- 検索（サイドバーの検索ボックスでタイトル・本文を部分一致フィルタ）
- 新規作成後は自動的にそのページへ遷移（`docRef.id` を使って `location.hash` を書き換え）
- **フォルダによるページ一覧の階層表示**（2026-07-05実装、当初はタイトルの「/」区切りで実装したが「毎回区切りを入れるのが面倒」というフィードバックを受けて全面刷新）
  - 新しいFirestoreコレクション `wiki_folders`（`name` `parentId`（親フォルダID、ルート直下なら`null`）`uid` `createdAt`）を追加。フォルダは多階層（フォルダの中にフォルダ）に対応
  - `wiki_pages` に `folderId` フィールドを追加（未所属なら`null`）。ページ作成・編集フォームの「フォルダ」ドロップダウンで所属フォルダを選択する
  - サイドバーの「＋ 新規フォルダ」ボタン（`#/new-folder`）でフォルダ名と親フォルダを指定して作成
  - `buildFolderTree()` が `wiki_folders` と `wiki_pages` を組み合わせて木構造化 → `renderFolderNode()` が `<details>/<summary>` の入れ子で再帰描画（フォルダ開閉はネイティブ、JS操作不要）
  - フォルダ削除（サイドバーの各フォルダの「×」）は、中にページ・サブフォルダが残っている場合はブロックする（空にしてからのみ削除可、権限は投稿者に限定せず誰でも削除できる＝タグと同じ共同編集ポリシー）
  - 検索・タグ絞り込み中は該当ページが見えるよう全フォルダを強制的に開き、該当なしのフォルダは非表示にする（`forceOpen` + `nodeHasVisibleContent()`）。それ以外は選択中ページを含むフォルダだけ自動で開く（`subtreeContainsSelected()`）
  - 記事詳細側にもパンくずリストを追加（`renderBreadcrumb()`）。`folderId`の親チェーンをたどって表示（フォルダ自体はページではないためリンクなし、テキスト表示のみ）

## 再開時のチェックポイント

1. このファイルで現状把握
2. `git log` で直近のコミット確認
3. 動作確認は `index.html` を `file://` で直接開いても**Googleログインができない**
   （Firebase AuthのOAuthポップアップはhttp/httpsオリジンが必須）。
   リポジトリルートで `python -m http.server 8080` を起動し、
   `http://localhost:8080/index.html` で確認する
4. 本番公開ドメイン（`kazuki-murasawa.github.io`）でログインさせるには、
   Firebase Console → Authentication → Settings → 承認済みドメイン に
   そのドメインを追加登録しておく必要がある（`localhost`はデフォルトで登録済み）
